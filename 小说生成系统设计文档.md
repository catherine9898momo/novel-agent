# 小说生成系统 - 项目设计文档

## 项目概述

基于 Agent 协作架构的智能小说生成系统，能够批量生成文笔统一、情节连贯、文学质量优秀的长篇小说。系统采用多 Agent 协作模式，模拟真实的文学创作团队工作流程。

## 核心目标

1. **文笔统一性**：通过风格 Agent 和编辑 Agent 确保整部作品的语言风格一致
2. **情节质量**：使用剧情 Agent 和审查 Agent 保证故事逻辑性和吸引力
3. **批量生成**：支持并行处理多部小说，提高生产效率
4. **可扩展性**：模块化设计，易于添加新的文学流派和写作风格

## Agent 概念映射

### s01: Agent Loop - 核心引擎
每个 Agent 都基于标准的 Agent Loop 运行：
- 接收任务 → 调用 LLM → 执行工具 → 返回结果 → 循环

**应用场景**：
- 写作 Agent 循环生成章节内容
- 编辑 Agent 循环修改润色文本

### s02: Tool Dispatch - 工具系统
为不同 Agent 配置专用工具集：

```
剧情 Agent 工具:
  - plot_outline: 生成故事大纲
  - character_design: 设计角色
  - conflict_check: 检查情节冲突

写作 Agent 工具:
  - write_chapter: 写作章节
  - style_apply: 应用文风模板
  - dialogue_generate: 生成对话

编辑 Agent 工具:
  - grammar_check: 语法检查
  - style_unify: 统一文风
  - polish: 润色文本

质量审查 Agent 工具:
  - logic_check: 逻辑审查
  - quality_score: 质量评分
  - feedback_generate: 生成反馈
```

### s03: TodoWrite - 进度跟踪
每个 Agent 维护自己的任务列表：

```
写作 Agent 的 Todo:
[x] 第1章：开篇场景
[>] 第2章：主角登场
[ ] 第3章：冲突引入
[ ] 第4章：情节发展
...
```

**实现细节**：
- 每完成一个章节，自动更新进度
- Lead Agent 可以实时查看所有 Agent 的进度
- 用户界面显示整体创作进度条

### s04: Subagent - 子任务隔离
复杂任务使用 Subagent 处理：

```
主任务: 生成第5章
  ↓
  启动 Subagent: 角色心理描写
    - 独立上下文
    - 专注于心理活动
    - 返回精炼的心理描写片段
  ↓
  整合到章节中
```

**应用场景**：
- 角色对话生成
- 场景细节描写
- 专业领域知识查询（如武侠小说的武功描写）

### s05: Skill Loading - 风格库
按需加载不同的写作风格和流派知识：

```
System Prompt (精简):
"你是一个小说写作 Agent
 可用风格: 武侠、言情、科幻、悬疑、历史
 需要时调用 load_style 获取详细写作指南"

当需要写武侠小说时:
  load_style("武侠")
  ↓
  返回:
  - 武侠小说的语言特点
  - 常用意象和修辞
  - 武打场面描写技巧
  - 江湖规矩和术语
  - 经典范例参考
```

**风格库结构**：
```
skills/
  styles/
    wuxia.md        # 武侠风格
    romance.md      # 言情风格
    scifi.md        # 科幻风格
    mystery.md      # 悬疑风格
  techniques/
    dialogue.md     # 对话技巧
    description.md  # 描写技巧
    pacing.md       # 节奏控制
```

### s06: Context Compact - 长篇处理
处理长篇小说时的上下文管理：

```
写作第50章时:
  - 保留: 最近3章的完整内容
  - 压缩: 第1-47章的摘要
  - 持久化: 完整内容存储在文件中

压缩策略:
  Layer 1: 每章完成后，旧章节内容 → "[第X章: 简要摘要]"
  Layer 2: 每10章触发一次大纲重整
  Layer 3: Agent 可主动调用 compact 工具
```

**实现细节**：
```
.context/
  novel_001/
    full_text.txt      # 完整文本
    chapter_summary.json  # 章节摘要
    character_state.json  # 角色状态追踪
    plot_timeline.json    # 情节时间线
```

### s07: Task System - 创作任务管理
将小说创作分解为持久化任务：

```
.tasks/
  novel_001/
    task_001.json  # 大纲创作
    task_002.json  # 角色设定
    task_003.json  # 第1章写作
    task_004.json  # 第1章编辑
    task_005.json  # 第1章审查
    ...
```

**任务状态流转**：
```
pending → assigned → in_progress → review → completed
                                      ↓
                                   rejected → in_progress
```

**任务依赖关系**：
```json

  "chapter": 1,

  "assignedTo": "writer_agent_1"

```

### s08: Background Tasks - 后台生成
长时间的生成任务在后台执行：

```
用户: "生成一部30万字的武侠小说"
  ↓
Lead Agent:
  1. 创建后台任务
  2. 立即返回 task_id
  3. 用户可以继续做其他事
  ↓
后台线程:
  - 生成大纲 (5分钟)
  - 生成角色 (3分钟)
  - 生成100章 (2小时)
  ↓
完成后通知用户
```

**后台任务类型**：
- 大纲生成
- 批量章节生成
- 全文润色
- 质量审查

### s09: Agent Teams - 创作团队
模拟真实的文学创作团队：

```
Lead Agent (总编辑)
  ↓
  ├── Plot Agent (剧情策划)
  │     - 负责故事大纲
  │     - 情节设计
  │     - 冲突设置
  │
  ├── Writer Agent × 3 (写作组)
  │     - 并行写作不同章节
  │     - 遵循统一风格指南
  │     - 互相参考已完成章节
  │
  ├── Editor Agent (编辑)
  │     - 统一文风
  │     - 语法润色
  │     - 细节完善
  │
  └── QA Agent (质量审查)
        - 逻辑检查
        - 质量评分
        - 提出修改建议
```

**协作流程**：
```
1. Plot Agent 生成大纲 → 提交审批
2. Lead Agent 审批通过 → 分配章节
3. Writer Agents 并行写作 → 提交初稿
4. Editor Agent 润色 → 提交修改版
5. QA Agent 审查 → 反馈问题
6. 循环 4-5 直到通过
7. Lead Agent 整合成书
```

### s11: Autonomous Agents - 自主创作
Writer Agents 自主认领任务：

```
Writer Agent 生命周期:
  启动 → 检查任务板 → 认领任务 → 写作 → 提交 → 空闲
    ↑                                              ↓
    └──────────────── 继续认领 ←──────────────────┘
                          ↓
                    空闲60秒 → 自动关闭

任务板:
  .tasks/available/
    chapter_005.json  # 未认领
    chapter_006.json  # 未认领
    chapter_007.json  # 已被 writer_1 认领
```

**自主决策**：
- Writer Agent 根据自己的负载选择任务
- 优先认领与自己已完成章节相邻的任务（保持连贯性）
- 如果发现质量问题，主动请求 Editor Agent 协助

### s12: Worktree Isolation - 多小说并行
使用目录隔离处理多部小说的并行生成：

```
主目录: /novel-generator
  │
  ├── .worktrees/
  │   ├── novel_001_武侠/
  │   │   ├── outline.md
  │   │   ├── chapters/
  │   │   └── .tasks/
  │   │
  │   ├── novel_002_言情/
  │   │   ├── outline.md
  │   │   ├── chapters/
  │   │   └── .tasks/
  │   │
  │   └── novel_003_科幻/
  │       ├── outline.md
  │       ├── chapters/
  │       └── .tasks/
  │
  └── 各小说独立工作，互不干扰
```

**隔离优势**：
- 不同小说的 Agent 团队完全独立
- 避免风格混淆
- 支持真正的并行生成

## 技术实现

### 核心模块

#### 1. Agent 引擎
```typescript
// 伪代码
class NovelAgent {
  role: 'plot' | 'writer' | 'editor' | 'qa'
  tools: ToolRegistry
  context: MessageHistory

  async run(task: Task): Promise<Result> {
    while (true) {
      const response = await this.callLLM()
      if (!response.toolCalls) break

      for (const call of response.toolCalls) {
        const result = await this.tools.execute(call)
        this.context.push(result)
      }
    }
    return response.text
  }
}
```

#### 2. 任务调度器
```typescript
class TaskScheduler {
  async assignTask(task: Task): Promise<void> {
    // 根据任务类型分配给合适的 Agent
    const agent = this.selectAgent(task.type)
    await agent.run(task)
  }

  async monitorProgress(): Promise<Progress> {
    // 收集所有 Agent 的进度
    return this.aggregateProgress()
  }
}
```

#### 3. 风格管理器
```typescript
class StyleManager {
  async loadStyle(genre: string): Promise<StyleGuide> {
    // 从技能库加载风格指南
    return await readFile(`skills/styles/${genre}.md`)
  }

  async unifyStyle(chapters: Chapter[]): Promise<void> {
    // 统一多个章节的文风
    const guide = await this.loadStyle(this.currentGenre)
    for (const chapter of chapters) {
      await this.editorAgent.polish(chapter, guide)
    }
  }
}
```

#### 4. 质量控制器
```typescript
class QualityController {
  async review(chapter: Chapter): Promise<QAReport> {
    // 多维度质量检查
    return {
      logicScore: await this.checkLogic(chapter),
      styleScore: await this.checkStyle(chapter),
      grammarScore: await this.checkGrammar(chapter),
      suggestions: await this.generateFeedback(chapter)
    }
  }
}
```

## 工作流程

### 完整创作流程

```
1. 项目初始化
   ├── 用户输入：题材、风格、长度
   ├── Lead Agent 创建项目结构
   └── 加载对应风格库

2. 大纲创作阶段
   ├── Plot Agent 生成故事大纲
   ├── 设计主要角色
   ├── 规划情节走向
   └── Lead Agent 审批

3. 章节分配阶段
   ├── 将大纲分解为章节任务
   ├── 创建任务依赖关系
   └── 发布到任务板

4. 并行写作阶段
   ├── Writer Agents 自主认领任务
   ├── 参考风格指南写作
   ├── 实时同步角色状态
   └── 提交初稿

5. 编辑润色阶段
   ├── Editor Agent 统一文风
   ├── 语法和用词优化
   ├── 细节完善
   └── 提交修改版

6. 质量审查阶段
   ├── QA Agent 逻辑检查
   ├── 质量评分
   ├── 生成反馈报告
   └── 不合格章节退回修改

7. 整合发布阶段
   ├── Lead Agent 整合所有章节
   ├── 生成完整小说文件
   ├── 生成元数据和目录
   └── 输出最终作品
```

## 质量保证机制

### 1. 文笔统一性保证
- **风格模板**：所有 Writer Agents 共享同一份风格指南
- **参考机制**：写作时参考已完成章节的文风
- **编辑审查**：Editor Agent 专门负责统一文风
- **对比检测**：自动检测文风偏离度

### 2. 情节连贯性保证
- **时间线追踪**：维护完整的情节时间线
- **角色状态管理**：实时同步角色的位置、状态、关系
- **冲突检测**：自动检测情节矛盾
- **逻辑审查**：QA Agent 专门检查逻辑问题

### 3. 质量评分体系
```
综合质量分 =
  文笔质量 × 0.3 +
  情节质量 × 0.3 +
  逻辑严密性 × 0.2 +
  创新性 × 0.1 +
  可读性 × 0.1

阈值设置:
  优秀: ≥ 0.9
  良好: ≥ 0.8
  合格: ≥ 0.7
  需修改: < 0.7
```

### 4. 迭代优化机制
```
章节评分 < 阈值
  ↓
生成详细反馈
  ↓
Writer Agent 修改
  ↓
Editor Agent 润色
  ↓
QA Agent 重新评分
  ↓
循环直到达标（最多3轮）
```

## 风险与挑战

### 1. 技术风险
- **上下文限制**：超长小说可能超出 LLM 上下文窗口
  - 缓解：分层压缩 + 智能摘要
- **质量波动**：不同章节质量可能不稳定
  - 缓解：多轮审查 + 质量阈值

### 2. 创作风险
- **情节重复**：可能出现套路化情节
  - 缓解：创新性检测 + 多样性约束
- **角色崩坏**：长篇中角色性格可能不一致
  - 缓解：角色状态追踪 + 性格一致性检查

### 3. 性能风险
- **生成时间长**：30万字小说可能需要数小时
  - 缓解：后台任务 + 并行化
- **成本较高**：大量 API 调用成本不低
  - 缓解：缓存复用 + 模型选择优化

## 总结

本系统通过将 Agent 概念应用于文学创作领域，实现了：

1. **模块化设计**：每个 Agent 职责明确，易于维护和扩展
2. **协作机制**：多 Agent 协作模拟真实创作团队
3. **质量保证**：多层审查机制确保作品质量
4. **高效生成**：并行化和后台任务提高效率
5. **可扩展性**：易于添加新流派和功能

这是一个将 AI Agent 技术与文学创作深度结合的创新项目，具有很强的实用价值和技术示范意义。
