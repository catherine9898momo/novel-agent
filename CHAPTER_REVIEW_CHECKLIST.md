# 章节评审清单（Chapter Review Checklist）

> 用途：写完每章后，先过清单再进入正式评审。
> 原则：每条都可验证，不能写“感觉还行”。

## A. 情节与结构
- [ ] 本章目标是否明确（不是“发生了很多事”）
- [ ] 每个核心场景是否有信息增量或关系变化
- [ ] 本章结尾是否留下明确钩子（情节或情绪）

## B. 人物与对话
- [ ] 主要角色说话是否可区分（词汇、句长、语气）
- [ ] 每段对话是否至少完成一项功能：推进关系 / 暴露性格 / 制造信息差
- [ ] 是否存在“说设定”的解释型台词（有则删或改为动作）

## C. 文笔与情绪
- [ ] 是否出现模板化陈词（如“温润如玉”“月色如水”等）
- [ ] 是否出现直白心理说明（如“她心想/他觉得”）
- [ ] 情绪转折是否有前置铺垫（避免突兀跳变）

## D. 节奏与转场
- [ ] 场景切换前是否有情绪或环境过渡句
- [ ] 段落长度是否有变化（避免全章同长度段落）
- [ ] 高张力段落是否用更短句加速

## E. 连贯性与伏笔
- [ ] 称谓、时间线、空间位置是否一致
- [ ] 本章伏笔状态是否更新（埋下/推进/回收）
- [ ] 本章内容是否与 `_story_so_far.md` 一致

## F. 交付动作
- [ ] 已保存章节文件
- [ ] 已更新 `_story_so_far.md`
- [ ] 已写 `_handoff_xxx.md`
- [ ] 已更新 `_foreshadowing.json`
