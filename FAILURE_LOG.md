# 失败复盘日志（Failure Log）

> 目的：避免同类错误重复发生。
> 规则：每条复盘必须包含“现象 → 根因 → 修复 → 防复发机制”。

## 模板

### [日期] [章节/任务]
- 现象：
- 根因：
- 修复：
- 防复发机制（放在哪一层）：
  - Plan / Writer / Reviewer / Metrics / Workflow
- 验证结果：

---

## 示例（可替换）

### 2026-04-13 第1章《入京》
- 现象：对话可读但“说设定”较多，关系推进不足。
- 根因：章节计划只定义了场景，没有定义对话目标与潜台词。
- 修复：Writer 增加对话硬约束；Reviewer 新增 `dialogue_quality` 维度。
- 防复发机制（放在哪一层）：
  - Writer：对话必须完成“推进关系/暴露性格/制造信息差”之一。
  - Reviewer：维度评分中强制评估对话质量。
  - Metrics：增加禁忌词与直白心理描写密度检测。
- 验证结果：待下一章评分回归验证。
